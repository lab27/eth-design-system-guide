ETH Komponenten-Bibliothek (Web Components)

Der Komponenten-Bibliothek Ansatz

Diese Herangehensweise bietet eine fokussierte Sammlung von Framework-unabhängigen Komponenten, die in jedem Web-Projekt verwendet werden können. Wir konzentrieren uns auf die wichtigsten Markenelemente und Interaktionsmuster der ETH Zürich und stellen sie als Web Components bereit.

Diese Komponenten-Bibliothek befindet sich in aktiver Entwicklung. Wir konzentrieren uns auf Kernkomponenten und erweitern die Bibliothek schrittweise basierend auf dem Feedback der ETH-Community.

Schlüsselmerkmale

  • Web Components: Funktionieren in jedem Framework (React, Vue, Angular) oder ohne Framework
  • Fokussierter Umfang: Enthalten nur die wichtigsten, markenrelevanten UI-Elemente
  • Zukunftssicher: Basierend auf Webstandards
  • Barrierefreiheit: Erfüllt WCAG 2.1 AA-Standards
  • Responsive Design: Optimiert für alle Bildschirmgrößen

Schnellstart

1. Installation

Bash 
npm install @ethzurich/web-components

2. Import und Registrierung

JavaScript 
// In Ihrer JavaScript-Datei
import '@ethzurich/web-components';

3. Verwendung in HTML

HTML 
<!-- In Ihrem HTML -->
<eth-button variant="primary">ETH Zürich Button</eth-button>

Vorschau
Code
HTML 
<eth-button variant="primary">Primär</eth-button>
<eth-button variant="secondary">Sekundär</eth-button>
<eth-button disabled>Deaktiviert</eth-button>

<!-- Mit Icon -->
<eth-button variant="primary">
  <eth-icon name="plus"></eth-icon>
  Mit Icon
</eth-button>

<!-- Icon-Button -->
<eth-button variant="secondary" size="icon-only">
  <eth-icon name="plus"></eth-icon>
</eth-button>

Eigenschaften

  • variant: "primary" (Standard), "secondary", "text"
  • size: "small", "medium" (Standard), "large", "icon-only"
  • disabled: Boolean
  • href: Optional, um den Button als Link zu rendern
  • icon: Optional, um ein Icon innerhalb des Buttons anzuzeigen
  • iconPosition: "start" (Standard), "end"

Ressourcenschonender Ansatz

Unsere Komponenten-Bibliothek ist gezielt für Teams mit begrenzten Ressourcen konzipiert:

  • Fokussierter Umfang: Wir konzentrieren uns nur auf die wichtigsten, markenrelevanten Komponenten
  • Framework-unabhängig: Eine Bibliothek für alle Projekte, unabhängig vom verwendeten Framework
  • Einfache Wartung: Web Components basieren auf Webstandards und erfordern weniger Updates
  • Selektive Nutzung: Entwickler können nur die Komponenten importieren, die sie tatsächlich benötigen
  • AI-unterstützte Entwicklung: Wir nutzen KI-Tools für Codegenerierung und Tests

Nächste Schritte und Roadmap

Die Komponenten-Bibliothek wird kontinuierlich erweitert. In den kommenden Monaten planen wir:

  • Formular-Komponenten (Eingabefelder, Checkboxen, Radiobuttons)
  • Navigation (Header, Footer, Breadcrumbs)
  • Responsive Tabellen
  • Verbesserte Dokumentation und mehr Beispiele

Für Fragen, Feedback oder Vorschläge kontaktieren Sie bitte das ETH Design-System-Team unter design-system@ethz.ch.
